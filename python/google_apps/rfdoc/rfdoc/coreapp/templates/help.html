  • Q: How to search keyword by name?
    A: Inputing lower-case string can search out all the keywords which name contains the input string in all libraries.
    keyword name should match the whole string, not one of split word.
    e.g. "get units",
    can match "Get Units Pair By Plugin Unit" and "Get Units By Rcj",
    but can not match "Get spare Units."

  • Q: How to search keyword by MML?
    A: Input upper-case MML command, all the relative keywords will be displayed.
    e.g. ZUSI can search out all the commands which contain ZUSI command.

  • Q: How to search DMX keyword by STE name?
    A: Input upper-case STE name can search out all the specified DMX commands.
    e.g. "POM" "RCJ"

  • Q: How to search Chorus keyword by command?
    A: Inputing "STE:prefix" can search out all the specified chorus commands.
    e.g. "STE:arun" can match "Start Rmd Register Port Name"

  • Q: What's means the '@{xx}' suffix in keyword name? (e.g. Add Default Active Unit For Revertive Switchover @{cb_9_9_15} )
    A: '@{version_RNC|MGW}' means that keyword use 'keyword wrapper' and may have many versions.
    Use the keyword name before @ in test case, IpaMml library will find out the most suitable keyword for you.

  • Q: How to deprecated a keyword?
    A: add '*DEPRECATED*' in the doc of keyword. RFdoc will searching the doc to generate deprecated keyword list.

  • Q: When the keyword doc be updated?
    A: There is a hudson project to check IpaMml/Testcase svn per 15 minutes. the RFdoc will updated if the SVN have new change. you can trigger the updating task by clicking following link manually too.

    Update IpaMml doc now   Update test case resources doc now  

    Note: The updating action will starting in the backgroupd, please click here to view the process of doc updating.

  • Q: How to search test case as example by keyword?
    A: Search a keyword, click the Example: Search In Testcases link to get test cases list used this keyword.
